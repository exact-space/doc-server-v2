BHEL- Manual

Server settings

Bhel has 4 servers which are further split into VMs internally which are orchestrated as swarm clusters. Each services run as docker services in these swarms

  1. base-node-1

    This has a VM called appdb-1 where mongodb runs

  2. base-node-2

  3. base-node-3

  4. base-node-4

    Kairos and Scylla runs here

Plotting Metrics in Kairos

In Kairos portal, the health monitoring parameters are added as metrics and the values can be obtained by plotting those metrics. Metrics can be plotted based on each node in the following format

metric:<node>:<param>

For example:

metric:base-node-1:cpu - CPU usage in base-node-1

metric:base-node-1:disk - Disk usage in base-node-1

metric:appdb-1:mongodb - MongoDB status

metric:base-node-1:memory - RAM usage in base-node-1

SSH to VMs

Each VMs in the base nodes can be accessed using the ssh command as following 

{ "cmd": "ssh -i /home/expadmin/key/expkey appdb-1 'df -h'" }

Getting latest code

To make any changes in the BHEL services, first download the code from BHEL to the local machine, make necessary changes and then deploy back in BHEL. Zip the service whichever you want to download. Here in the below Request body change the data-api with whatever service you need. 

xxxxxxxxxx
{"cmd":"TS=$(date +%s) cd ~/es-master/src && zip -r filename-$TS.zip filename/ && cp -v filename-$TS.zip ~/es-master/src/uploads/docker/ && echo filename-$TS.zip"}

 

In case of UI code cd to the UI folder(cd ~/es-master/src/ui) and then zip the code in the above command.

This will generate a zip file.

Download the code

Specify the name of the zip file you generated in step-1.

xxxxxxxxxx
wget http://rmds.bhel.in/exactapi/attachments/docker/download/file-to-download.zip

Modifying the code.

After making the appropriate changes to the code compress it as a zip file using the command zip -r file.zip folder/.

Place this zip file under the location /space/es-master/bin/ in the prod server .

Execute ./code-deploy.py name-of-zip-to-upload.zip to upload it to remote server.

Deploy the service

Using the above URL pass the below payload to deploy the service. Here replace dataapi-es with the docker container name of the service that you want to deploy. Call the process2 URL with the for each request body below.

xxxxxxxxxx
{"cmd": "cp ~/es-master/src/uploads/docker/data-api.zip ~/es-master/src/  && cd ~/es-master/src/ && unzip -o data-api.zip"}
xxxxxxxxxx
{"cmd": "docker --tlsverify --tlscacert=/home/expadmin/.docker/ca.pem --tlscert=/home/expadmin/.docker/cert.pem --tlskey=/home/expadmin/.docker/key.pem -H vml-2:2376 service scale dataapi-es=1"}
xxxxxxxxxx
{"cmd": "cd ~/es-master/build.prod/data-api && ./stop_service.sh && VER=v1 ./build.sh && ./run_service.sh"}
xxxxxxxxxx
{"cmd": "docker --tlsverify --tlscacert=/home/expadmin/.docker/ca.pem --tlscert=/home/expadmin/.docker/cert.pem --tlskey=/home/expadmin/.docker/key.pem -H vml-2:2376 service scale dataapi-es=4"}

Reverting the build

Access log

Pass the below body to the process2 API to move the log file to the uploads location. This file can be either downloaded or can be accessed directly through the browser.

Note: Convert the log file to a txt file to access in browser.

xxxxxxxxxx
{"cmd": "cd /mnt/hdd1/log/data-api && tail -f kairos.log > ~/es-master/src/uploads/docker/kairos.txt"}

Cancel the request after 10 seconds as the tailing will be carried on the console. After doing this check the logs here: http://rmds.bhel.in/exactapi/attachments/docker/download/kairos.txt

  2. rm -rf app_alpha.zip
  3. rm -rf app_alpha_bk
  4. cp -rf app_alpha app_alpha_bk/
  5. cp -rf /space/es-master/src/ui/staging_test/app_alpha .

Troubleshoot guide

Captcha service

Use the below service to restart the captcha service

xxxxxxxxxx
{"cmd":"cd ~/es-master/build/captchas && ./run_service.sh"}

Docker

Restart docker service

xxxxxxxxxx
{"cmd": "sshpass -f pwd_file ssh expadmin@base-node-4 'sudo nohup dockerd >> a.out 2>&1 &'"}

Kairos

Stop and start Kairos

xxxxxxxxxx
{"cmd": "sshpass -f pwd_file ssh expadmin@base-node-4 'cd /opt/kairosdb/bin && sudo ./kairosdb-service.sh stop'"}
xxxxxxxxxx
{"cmd": "sshpass -f pwd_file ssh expadmin@base-node-4 'cd /opt/kairosdb/bin && sudo ./kairosdb-service.sh start'"}
